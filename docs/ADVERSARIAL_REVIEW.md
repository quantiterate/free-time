# Adversarial Judge Review

## Strongest case for an award
The concept is differentiated, emotionally legible, technically defensible, and broadly useful. It combines an exact time-box with whole-person suitability and explainable uncertainty.

## Largest remaining weakness
The sample candidate database is curated. The demo proves the decision architecture, not comprehensive real-world venue coverage.

## Judge attacks and responses

### “This is just a chatbot.”
Response: deterministic code enforces time, return buffer, mode, accessibility, sensory, walking, seating, and pet rules. GPT-5.6 cannot overrule them.

### “This stereotypes health conditions.”
Response: recommendations use functional and user-stated constraints, not diagnosis-to-activity mappings.

### “The ML is fake.”
Response: the repository contains an explicit cosine-similarity nearest-neighbor model and feedback-driven profile updates.

### “The data is unreliable.”
Response: every result carries confidence and uncertainty. Production adapters would retain provenance and timestamps.

### “The scope is too broad.”
Response: the submission demonstrates one complete vertical slice with three personas and curated candidates.
